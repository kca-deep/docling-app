# 추천 질문 생성

RAG 시스템의 추천 질문을 생성합니다.
일반 직원이 업무 중 실제로 궁금해할 만한 실용적인 질문을 생성하세요.

## 입력 정보
- 컬렉션: {collection_name}
- 문서 유형: {template_type}
- 문서 샘플:
{document_sample}

## CRITICAL RULES (MUST FOLLOW)
1. **문서에서 확실히 답변 가능한 질문만 생성** - 추측이 필요한 질문 금지
2. 문서 샘플에 명시된 키워드/개념을 기반으로 질문 생성
3. **일반 직원 관점의 실용적 질문 생성** - 실제 업무에서 물어볼 법한 질문
4. 너무 일반적인 질문 금지 (예: "이게 뭐예요?", "알려주세요")
5. 문서에 없는 내용에 대한 질문 생성 금지
6. **신뢰도 높은 답변이 가능한 질문 우선** - 문서에서 명확하게 답변할 수 있는 질문 생성

## 피해야 할 질문 패턴 (AVOID)
다음 패턴은 학술적/형식적이므로 사용하지 마세요:
- "~의 정의는 무엇인가요?" / "~란 무엇인가요?"
- "~의 종류는 무엇인가요?" / "~유형은 무엇인가요?"
- "~와 ~의 차이점은 무엇인가요?"
- "~의 적용 범위는 무엇인가요?"
- "~의 개념을 설명해주세요"

## 권장 질문 패턴 (PREFERRED)

### 신청/이용 관련 (가장 중요)
- "~하려면 어떻게 해야 하나요?"
- "~신청 절차는 어떻게 되나요?"
- "~은 어떻게 하나요?"
- "~예약/이용은 어떻게 하나요?"

### 기준/방법 관련
- "~은 어떻게 되나요?"
- "~은 어떻게 계산되나요?"
- "~기준은 무엇인가요?"
- "~은 어떻게 진행되나요?"

### 조건/자격 관련
- "~조건은 무엇인가요?"
- "~할 수 있나요?"
- "~자격이 인정되나요?"
- "~대상은 누구인가요?"

### 혜택/수치 관련
- "~은 얼마까지 받을 수 있나요?"
- "~며칠까지 사용할 수 있나요?"
- "~기간은 어떻게 되나요?"
- "~언제까지 가능한가요?"

## 템플릿별 질문 패턴

### regulation (규정 문서)
- "~하려면 어떻게 해야 하나요?"
- "~신청 시 필요한 서류는 무엇인가요?"
- "~조건과 절차는 무엇인가요?"
- "~위반 시 어떤 처분을 받게 되나요?"

### budget (예산/회계)
- "~청구 절차는 어떻게 하나요?"
- "~사용 가능 범위는 어떻게 되나요?"
- "~에 필요한 증빙서류는 무엇인가요?"
- "~예약/이용은 어떻게 하나요?"

### fund (기금사업)
- "~정산 절차는 어떻게 되나요?"
- "~변경 신청은 어떻게 하나요?"
- "~집행 시 주의사항은 무엇인가요?"
- "불인정금액이 발생하면 어떻게 처리되나요?"

### technical (기술 문서)
- "~사용 방법은 어떻게 되나요?"
- "~설정/설치 절차는 무엇인가요?"
- "~오류가 발생하면 어떻게 해결하나요?"
- "~보안 점검 항목은 무엇인가요?"

### default/casual
- 문서 내용에 맞는 자연스러운 질문 형태 사용
- 직원이 실제로 물어볼 법한 실용적 질문

## 질문 유형 (4개 생성)

### 필수 유형 (3개)
1. **신청/이용** 1개: "~하려면 어떻게 해야 하나요?", "~은 어떻게 하나요?"
2. **기준/방법** 1개: "~은 어떻게 되나요?", "~기준은 무엇인가요?"
3. **조건/자격** 1개: "~조건은 무엇인가요?", "~할 수 있나요?"

### 선택 유형 (1개 - 문서 내용에 따라 선택)
- **혜택/수치**: "~얼마까지?", "~며칠까지?", "~기간은?" (수치가 문서에 있을 때)
- **서류/양식**: "~에 필요한 서류는?" (별지/서식이 있을 때)
- **주의/예외**: "~주의사항은?", "~위반 시?" (단서 조항이 있을 때)
- **제도설명**: "~제도가 무엇인가요?" (새로운 제도 소개 시에만)

## 품질 기준
- 질문 길이: 15~50자 권장
- 반드시 물음표(?)로 종결
- 문서 샘플의 핵심 키워드 1개 이상 포함
- **실무자가 업무 중 실제로 물어볼 법한 실용적 질문**

## 좋은 예시 vs 나쁜 예시

### 좋은 예시
- "연차휴가는 며칠까지 사용할 수 있나요?"
- "유연근무제를 신청하려면 어떻게 해야 하나요?"
- "초과근무수당은 어떻게 계산되나요?"
- "법인카드 사용 가능 범위는 어떻게 되나요?"

### 나쁜 예시
- "연차휴가의 정의는 무엇인가요?" (학술적)
- "초과근무수당의 종류는 무엇인가요?" (형식적)
- "연차휴가와 저축연가의 차이점은 무엇인가요?" (비교 질문)
- "인사규정의 적용 범위는 무엇인가요?" (너무 일반적)

## 규칙
1. 문서에서 답변 가능한 질문만 생성
2. **직원 관점의 구체적이고 실용적인 질문**
3. 각 질문은 물음표(?)로 끝낼 것
4. 4개 질문은 서로 다른 주제를 다룰 것
5. 피해야 할 패턴은 절대 사용하지 않을 것

## 출력 형식 (필수)
- 반드시 JSON 문자열 배열만 출력
- 코드 블록(```) 없이 바로 출력
- 설명이나 번호 목록 없이 배열만 출력

올바른 출력 예시:
["동호회 활동 지원금은 얼마까지 받을 수 있나요?", "콘도 이용 예약은 어떻게 하나요?", "자녀 학자금 대부를 신청하려면 어떤 조건이 필요한가요?", "사택 입주 신청 절차는 어떻게 되나요?"]
