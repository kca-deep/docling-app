# 추천 질문 생성

RAG 시스템의 추천 질문을 생성합니다.

## 입력 정보
- 컬렉션: {collection_name}
- 문서 유형: {template_type}
- 문서 샘플:
{document_sample}

## CRITICAL RULES (MUST FOLLOW)
1. **문서에서 확실히 답변 가능한 질문만 생성** - 추측이 필요한 질문 금지
2. 문서 샘플에 명시된 키워드/개념을 기반으로 질문 생성
3. 너무 일반적인 질문 금지 (예: "이게 뭐예요?", "알려주세요")
4. 너무 구체적인 질문 금지 (예: 특정 날짜/금액을 직접 묻는 질문)
5. 문서에 없는 내용에 대한 질문 생성 금지
6. **신뢰도 높은 답변이 가능한 질문 우선** - 문서에서 명확하게 답변할 수 있는 질문 생성

## 템플릿별 질문 패턴

### regulation (규정 문서)
- "~의 적용 대상은 누구인가요?"
- "~하려면 어떤 절차를 거쳐야 하나요?"
- "~의 조건/요건은 무엇인가요?"
- "~의 예외 조항은 무엇인가요?"

### budget (예산/회계)
- "~비용 청구 절차는 어떻게 되나요?"
- "~의 지급 기준은 무엇인가요?"
- "~에 필요한 증빙서류는 무엇인가요?"
- "~와 ~의 차이점은 무엇인가요?"

### fund (기금사업)
- "~정산 절차는 어떻게 되나요?"
- "~변경 신청 방법은 무엇인가요?"
- "~지원 조건은 무엇인가요?"
- "~의 제출 서류는 무엇인가요?"

### technical (기술 문서)
- "~의 사용 방법은 어떻게 되나요?"
- "~설정/설치 절차는 무엇인가요?"
- "~오류 해결 방법은 무엇인가요?"
- "~의 요구사항은 무엇인가요?"

### default/casual
- 문서 내용에 맞는 자연스러운 질문 형태 사용

## 질문 유형 (4개 생성)

### 필수 유형 (3개)
1. **정의/개념** 1개: "~란?", "~의 정의는?", "~이 무엇인가요?"
2. **절차/방법** 1개: "~하려면?", "~절차는?", "~방법은?"
3. **조건/요건** 1개: "~조건은?", "~가능한 경우는?", "~자격은?"

### 선택 유형 (1개 - 문서 내용에 따라 선택)
- **비교/차이**: "~와 ~의 차이점은?" (비교 대상이 문서에 있을 때)
- **계산/금액**: "~는 얼마인가요?", "~기간은?" (수치가 문서에 있을 때)
- **예외/주의**: "~의 예외사항은?", "~주의사항은?" (단서 조항이 있을 때)
- **서류/양식**: "~에 필요한 서류는?" (별지/서식이 있을 때)

## 품질 기준
- 질문 길이: 15~50자 권장
- 반드시 물음표(?)로 종결
- 문서 샘플의 핵심 키워드 1개 이상 포함
- 실무자가 실제로 물어볼 법한 실용적 질문

## 규칙
1. 문서에서 답변 가능한 질문만 생성
2. 구체적이고 실용적인 질문
3. 각 질문은 물음표(?)로 끝낼 것
4. 4개 질문은 서로 다른 주제를 다룰 것

## 출력 형식 (필수)
- 반드시 JSON 문자열 배열만 출력
- 코드 블록(```) 없이 바로 출력
- 설명이나 번호 목록 없이 배열만 출력

올바른 출력 예시:
["질문1?", "질문2?", "질문3?", "질문4?"]
