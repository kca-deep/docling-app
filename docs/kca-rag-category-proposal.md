## 1. 카테고리 설계 원칙

### 1.1 RAG 최적화 기준
- **질문 유형 기반 분류**: 사용자가 자주 묻는 질문 패턴에 맞춰 분류
- **업무 맥락 연결**: 관련 규정을 함께 검색할 수 있도록 그룹화
- **계층적 구조**: 상위 규정과 하위 세칙/지침을 연계
- **검색 정확도 향상**: 유사 주제 규정을 동일 컬렉션으로 구성

### 1.2 권장 컬렉션 구조
```
Collection per Category (권장)
├── 각 카테고리별 별도 Qdrant 컬렉션
├── 질문 유형에 따른 컬렉션 선택적 검색
└── 크로스 컬렉션 검색 지원
```

## 4. RAG 시스템 구성 권장사항

### 4.1 컬렉션 운영 방안

#### Option A: 통합 컬렉션 + 메타데이터 필터링 (단순)
```
단일 컬렉션: kca-regulations
메타데이터: { category: "kca-hr", sub_category: "채용" }
검색 시: 메타데이터 필터로 카테고리 제한
```

#### Option B: 카테고리별 분리 컬렉션 (권장)
```
14개 컬렉션 분리 운영
질문 의도 분류 후 해당 컬렉션 검색
복합 질문 시 다중 컬렉션 검색
```

### 4.2 검색 최적화 전략

| 전략 | 설명 |
|------|------|
| 질문 의도 분류 | LLM으로 질문 카테고리 먼저 분류 |
| 계층적 검색 | 상위규정 → 하위규칙 순서로 우선순위 부여 |
| 연관 규정 연결 | 같은 카테고리 내 관련 규정 함께 검색 |
| 키워드 부스팅 | 규정별 연관 키워드에 가중치 부여 |

### 4.3 시스템 프롬프트 예시

```
당신은 한국방송통신전파진흥원(KCA) 내부규정 전문가입니다.
사용자의 질문에 대해 관련 규정을 검색하여 정확하게 답변합니다.

답변 시 다음을 준수합니다:
1. 반드시 근거 규정명과 조항을 명시합니다.
2. 상위규정(규정)과 하위규칙(규칙/세칙/지침)을 구분합니다.
3. 관련된 다른 규정이 있으면 함께 안내합니다.
4. 규정에 명시되지 않은 사항은 추측하지 않습니다.
```
